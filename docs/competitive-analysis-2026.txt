1. РЕЗЮМЕ ДЛЯ РУКОВОДСТВА
================================================================================

ТЕХЗАКАЗ — НЕ конкурент B2B-Center и Фабриканта. Это другой продукт:
  • не тендерная ЭТП под 44-ФЗ / 223-ФЗ;
  • не каталог SKU «купить станок с полки»;
  • а платформа ПРЯМЫХ закупок по чертежу/ТЗ для нефтесервиса и смежной
    промышленности (РТИ, металл, арматура, электро).

Рынок ЭТП огромен (сотни тысяч процедур в год, лидеры — B2B-Center ~245 тыс.,
B2B-РТС ~41% коммерческих процедур), но там выигрывают деньги, регуляторика,
ЭЦП и 20 лет доверия. Наш путь — узкая вертикаль + скорость + полный цикл
сделки без тендерного цирка.

Главный прямой конкурент по смыслу продукта:
  • Supl.biz / OptList (доски RFQ) — массовость, но слабый цикл сделки;
  • Promchat / ROSINDUSTRY — «superapp промышленности», широкий, но размытый;
  • Rusoil Market — та же ниша нефтегаз, но мёртвый трафик (десятки участников).

Главное конкурентное преимущество ТехЗаказ СЕЙЧАС:
  чертёж → КП → сравнение → чат → поставка → отзыв — в одном продукте,
  с match-score, бенчмарком цен и 0% комиссии с контракта (launch).

Главный разрыв СЕЙЧАС:
  масштаб (1,8 млн поставщиков у Supl vs наш каталог + реестр),
  интеграции с ERP/SRM, бренд, юридическая инфраструктура (ЭЦП, договоры).

2. КОНТЕКСТ РЫНКА 2025–2026
================================================================================

По отраслевым отчётам (Data Insight, T-Bank Research, RAEX):

  • Коммерческий сегмент ЭТП растёт быстрее всего (~17% CAGR).
  • B2B-РТС — лидер регулируемых закупок (~38% рынка 44-ФЗ/223-ФЗ).
  • B2B-Center — лидер по числу процедур (~245 тыс./год), экосистема с РТС.
  • Тренды:
      — уход от дорогих «каптивных» корпоративных площадок к независимым;
      — рост SRM / eProcurement у крупняка;
      — закрытые площадки с предквалификацией поставщиков;
      — интеграции с ERP, аналитика, контроль исполнения контракта;
      — сокращение числа поставщиков → скоринг и предквалификация.

  • Для МСП и средних промышленных компаний остаётся незакрытая ниша:
      «мне нужно 200 манжет по чертежу за 14 дней, без тендера на 3 месяца».

Именно эту нишу занимает ТехЗаказ.

──────────────────────────────────────────────────────────────────────────────
4.1. B2B-CENTER + ГРУППА (РТС-ТЕНДЕР, ОТС)
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     600k+ заказчиков и поставщиков, «каждая вторая» коммерческая
               закупка крупняка (их заявление).
  Сильные:     тендеры 2.0, многоэтапные процедуры, ЭЦП, реестр ПО РФ,
               аналитика для корпораций, бренд №1.
  Слабые:      тяжёлый UX, ориентация на процедуру а не на сделку,
               чертёж/TZ — редкость, нет трекинга поставки «из коробки»,
               комиссии и абонплаты, долгий цикл.
  Для нас:     не биться лоб в лоб; забирать инженеров и снабженцев,
               которым «тендер на 3 млн за прокладки — overkill».

──────────────────────────────────────────────────────────────────────────────
4.2. B2B-РТС
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     ~38% регулируемых закупок, ~41% нерегулируемых процедур.
  Сильные:     госзакупки, доверие, IPO-история, полный стек 44/223/615.
  Слабые:      то же — процедура > сделка; не про изготовление по чертежу.
  Для нас:     игнорировать как прямого конкурента.

──────────────────────────────────────────────────────────────────────────────
4.3. ФАБРИКАНТ
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     с 2005 г., ~1,4 млн поставщиков, ~20k закупок/мес,
               16 трлн ₽ объём торгов (накопительно).
  Сильные:     корп. секции (Росатом, Роскосмос, Силовые машины),
               B2B-маркет без тендера, обучение, поддержка, RAEX топ-3.
  Слабые:      перегруженность для МСП; маркет = прайс-листы, не чертежи.
  Для нас:     перехватывать поставщиков, которым нужны «живые» RFQ по ТЗ,
               а не заливка YML.

──────────────────────────────────────────────────────────────────────────────
4.4. BIDZAAR
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     ~54 тыс. процедур/год (Data Insight), корпоративный SaaS.
  Сильные:     современный UX, кейсы (SPLAT −15%, Доброцен −35% срок),
               API, экосистема, фокус на непрямые закупки.
  Слабые:      тендерная логика; нет вертикали нефтесервис; нет match по
               специализации завода.
  Для нас:     конкурировать за «быструю закупку без комиссии отдела закупок».

──────────────────────────────────────────────────────────────────────────────
4.5. SUPL.BIZ
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     до 200k заказов/год, до 50 млрд ₽ GMV (з заявление),
               1,8 млн поставщиков в базе, 650 городов, бесплатно для покупателя.
  Сильные:     скорость размещения (1 мин), массовый охват, простота,
               нет ЭЦП — низкий порог входа, SEO и бренд в МСП.
  Слабые:      текстовая заявка без инженерного контекста; слабый чат/сделка;
               нет бенчмарка; нет трекинга поставки; «контакты и уходим в
               WhatsApp»; качество поставщиков неоднородно.
  Для нас:     ГЛАВНЫЙ конкурент по формату «разместил заявку — получил КП».
               Бить: чертёж, верификация, сравнение КП, сделка до конца.

──────────────────────────────────────────────────────────────────────────────
4.6. PROMCHAT
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     промышленная экосистема (закупки + логистика + перевозки).
  Сильные:     модульность, нефтегаз в списке отраслей, тендер + сравнение.
  Слабые:      размытое позиционирование; нет доминирования в одной вертикали;
               ранняя стадия узнаваемости.
  Для нас:     выиграть глубиной в нефтесервисе и UX конкретных сценариев.

──────────────────────────────────────────────────────────────────────────────
4.7. ROSINDUSTRY (SuperApp)
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     заявлено 500k+ товаров, база ЕГРЮЛ, маркетплейс «в наличии».
  Сильные:     проверка контрагентов, «всё в одном», безопасные сделки (обещание).
  Слабые:      каталог SKU ≠ изготовление по чертежу; широкий = shallow.
  Для нас:     EGRUL-верификацию уже делаем; добавить «claim профиля» из реестра.

──────────────────────────────────────────────────────────────────────────────
4.8. RUSOIL MARKET
──────────────────────────────────────────────────────────────────────────────
  Масштаб:     нишевый портал нефтегаза; на момент анализа — десятки участников,
               0 активных заявок на закупку (пустой marketplace).
  Сильные:     правильная ниша, понятная структура (товары/услуги/участники).
  Слабые:     нет трафика, нет network effect, устаревший контент.
  Для нас:     занять эту нишу первым с живым продуктом — «Rusoil мёртв,
               мы — рабочая альтернатива».

──────────────────────────────────────────────────────────────────────────────
4.9. ENEX / ПРОМХАБ (каталоги)
──────────────────────────────────────────────────────────────────────────────
  Суть:        SKU, остатки, «купить трубу DN150» — не кастомное производство.
  Для нас:     не конкурент; возможен SEO-трафик на стандартные позиции.

6. ГДЕ ТЕХЗАКАЗ ЛУЧШЕ КОНКУРЕНТОВ

6.1. ПРОДУКТОВЫЕ КЛИНЬЯ (то, чего нет у гигантов)

  ★ Закупка по чертежу/TZ как ядро
    Загрузка PDF/DWG/STEP, описание ГОСТ/допусков, AI-помощник ТЗ (DeepSeek).
    Supl — текст «нужны прокладки»; B2B-Center — процедура на 50 полей.

  ★ Полный цикл сделки в одном окне
    RFQ → КП → сравнение → выбор → чат → этапы поставки → отзыв.
    У Supl цикл обрывается на «связались и ушли»; у ЭТП — на «протокол
    подведения итогов».

  ★ Match-score с объяснением
    Поставщик видит «87% — горячий матч» и причины; email/push при ≥70%.
    Ни один массовый конкурент не делает семантический матч под специализацию
    завода так явно.

  ★ Бенчмарк цен по платформе
    Медиана/диапазон по категории в карточке закупки + публично на zakupki/*.
    Даёт заказчику аргумент «мы не переплачиваем» — редкость вне корп. SRM.

  ★ Сравнение КП + рекомендация платформы + PDF
    Взвешенный скоринг (цена 40%, срок 25%, верификация 15%, рейтинг 10%,
    match 10%) + таблица для согласования с директором.

  ★ Видимость загрузки производства
    freeCapacity / productionLoad в каталоге — «у завода 40% мощности свободно».
    Уникально для выбора срочных заказов.

  ★ Вертикаль нефтесервиса
    Категории, лендинг, SEO-страницы (РТИ, металл, арматура, электро),
    язык отрасли. Rusoil Market формально там же, но продукт мёртв.

  ★ Скорость и простота
    Без ЭЦП, без обеспечения заявки, без тендерного регламента — как Supl,
    но с инженерной глубиной.

  ★ Экономика для заказчика
    0% комиссии с контракта (launchMode) — сильный аргумент против ЭТП
    и маркетплейсов с %.

6.2. ТЕХНИЧЕСКИЕ ПЛЮСЫ (для зрелых пользователей)

  • Real-time чат (Socket.io), не только email
  • Web Push + email + (частично) Telegram
  • 2FA, сессии устройств (реальные refresh-tokens)
  • Экспорт Excel/PDF/1C CommerceML
  • Тёмная тема, mobile cards, command palette (Ctrl+K)
  • SEO-инфраструктура (GSC, Yandex Webmaster, schema.org на лендинге)
  • Автозакрытие заявок по deadline + напоминания

8. СТРАТЕГИЯ: КАК ИХ ОБОЙТИ (НЕ КОПИРОВАТЬ ГИГАНТОВ)
================================================================================

ПРИНЦИП: «Узкая вертикаль × полный цикл × скорость» — не «ещё одна ЭТП».

  ┌─────────────────────────────────────────────────────────────────┐
  │  НЕ ДЕЛАТЬ                         │  ДЕЛАТЬ                    │
  ├─────────────────────────────────────┼────────────────────────────┤
  │  Тендерный движок 223-ФЗ           │  Прямая закупка по чертежу │
  │  Каталог SKU миллиона позиций        │  Изготовление по ТЗ        │
  │  Конкуренция с B2B-Center по бренду  │  Доминировать в нефтесервис│
  │  Эскроу на старте                  │  Claim реестра + живые RFQ │
  │  Копировать Supl «всё для всех»      │  5 категорий — идеально    │
  └─────────────────────────────────────┴────────────────────────────┘

Три рычага роста:

  1. SUPPLY (поставщики)
     Реестр ГИСП → email invite → claim профиля → первое КП.
     Цель: конверсия заглушек в claimed + верифицированных.

  2. DEMAND (заказчики)
     SEO zakupki/* + лендинг + демо без регистрации + кейсы «−12% vs рынок».
     Цель: заказчик размещает RFQ с чертежом за <5 мин.

  3. STICKINESS (удержание)
     Сделка не уходит в WhatsApp: чат, трекинг, отзывы, повторный шаблон
     закупки, избранные поставщики, бенчмарк.

      Конкуренты: Supl (живая база). Мы: 4287 потенциальных.

  [2] Публичные кейсы + цифры на лендинге
      «Закупка РТИ: 4 КП за 36 ч, −12% к медиане» — даже 2–3 реальных кейса
      бьют Promchat и Rusoil.

  [3] Реальный API + webhook (убрать FAKE_API_KEY)
      • POST /api/v1/orders, GET /api/v1/proposals, webhook proposal:new
      • Bitrix24 / AmoCRM уже в settings — подключить по-настоящему
      Конкурент: Bidzaar API.

  [4] Telegram-бот до конца
      • новая закупка, новое КП, сообщение, смена этапа сделки
      Конкуренты: все шлют email; TG — наш канал для полевых инженеров.

  [5] SLA на профиле поставщика
      «Среднее время ответа: 4 ч», «% выигранных сделок» — публично.
      Supl этого не показывает; повышает доверие.

── P1: дифференциация (2–4 месяца) ───────────────────────────────────────

  [6] Двусторонняя интеграция 1С
      • входящие номенклатуры, исходящие заказы/КП
      B2B-Center сильны здесь — без 1C крупный снабженец не останется.

  [7] Согласование закупки внутри компании
      • «черновик → согласовал начальник → опубликовано»
      Bidzaar/SRM — есть; нам нужен light-version для среднего бизнеса.

  [8] Шаблоны ТЗ по категориям (без AI тоже)
      РТИ / арматура / металл — готовые поля ГОСТ, материал, допуски.
      AI-TZ поверх шаблона = быстрее и точнее.

  [9] «Пульс закупки» на карточке
      просмотры, новые КП, тренд цены vs бенчмарк — ощущение живой сделки.

  [10] PWA + push на mobile
       Снабженец на объекте — получил push «новое КП», открыл сравнение.

  [11] Типовой договор + спецификация (PDF/DOCX)
       Генерация из данных закупки — снижает friction после выбора КП.

── P2: доминирование в нише (4–8 месяцев) ───────────────────────────────

  [12] Закрытые площадки для холдинга
       «только мои верифицированные поставщики» — тренд Data Insight 2025.

  [13] Предквалификация поставщика
       чеклист: ISO, мощности, страховка, рейтинг → badge «Предквалифицирован».

  [14] Аналитика для закупщика v2
       «переплата vs бенчмарк», «лучший поставщик по срокам», weekly digest
       с инсайтами (не просто цифры).

  [15] Логистика (light)
       не Promchat-full, но поле «доставка / Incoterms / транспорт» в сделке.

  [16] Монетизация (после PMF)
       • freemium для заказчика
       • подписка поставщика: приоритет в match, больше RFQ, аналитика
       • НЕ % с контракта — наш текущий клин

  [17] Эскроу / оплата (опционально, с банком-партнёром)
       ROSINDUSTRY обещает «безопасные сделки» — если дойдём до крупных сумм.

── P3: не трогать до product-market fit ─────────────────────────────────

  • Полноценная ЭТП под 223-ФЗ
  • Мобильное native app (React Native)
  • Маркетплейс SKU «купить с полки»
  • Экспорт на СНГ / международный трафик

10. ПОЗИЦИОНИРОВАНИЕ И ОДНА ФРАЗА ДЛЯ РЫНКА
================================================================================

Целевой клиент (ICP):
  • снабженец / инженер / закупщик нефтесервисной или машиностроительной
    компании 50–500 человек;
  • закупка нестандартных изделий по чертежу 100k–30m ₽;
  • «тендер долго, Supl — слишком общий, Excel и WhatsApp — боль».

Целевой поставщик:
  • завод РТИ, металлообработки, арматуры с свободными мощностями;
  • хочет RFQ без тендерной бюрократии;
  • готов заполнить профиль и отвечать с файлом КП.

ОДНА ФРАЗА (маркетинг):
  «ТехЗаказ — прямые закупки по чертежам: загрузите ТЗ, получите КП от
   заводов, сравните цены и ведите сделку до поставки. Без тендера.
   Без комиссии с контракта.»

Против Supl:
  «Supl — доска объявлений. ТехЗаказ — инженерная закупка с чертежом,
   сравнением КП и контролем поставки.»

Против B2B-Center:
  «B2B-Center — когда нужен тендер на 50 млн. ТехЗаказ — когда нужны
   прокладки за 14 дней без отдела закупок.»

Против Rusoil Market:
  «Единственная живая площадка прямых закупок для нефтесервиса.»

11. РИСКИ И АНТИ-ПАТТЕРНЫ
================================================================================

  РИСК: распыление «сделаем superapp как Promchat»
  → держать фокус: чертёж → сделка → поставка

  РИСК: гонка за «1,8 млн поставщиков» как Supl
  → качество + верификация + match лучше, чем мёртвая база

  РИСК: registry invite спамит реальные заводы при тестах на prod
  → REGISTRY_INVITES_ENABLED=0 на staging; лимиты уже есть (20/закупку, 14 дней)

  РИСК: fake UI (API keys, старые заглушки сессий) подрывает доверие
  → убрать или доделать; частично исправлено (сессии — реальные)

  РИСК: конкуренты копируют match-score
  → moat = данные сделок + бенчмарк + отзывы + claimed реестр

  АНТИ-ПАТТЕРН: копировать тендерный UX B2B-Center
  АНТИ-ПАТТЕРН: брать % с контракта на старте (убьёт ICP)
  АНТИ-ПАТТЕРН: SEO на «тендеры 223-ФЗ» — не наш трафик

================================================================================
ИТОГ: 5 ШАГОВ, ЧТОБЫ «ЗАСТАВИТЬ ГЛОТАТЬ ПЫЛЬ»
================================================================================

  1. Захватить нишу нефтесервис + 5 категорий — SEO, кейсы, Rusoil obsolete.
  2. Оживить реестр — claim 500+ заводов с реальными профилями и КП.
  3. Удержать сделку в платформе — чат, трекинг, PDF, 1C, Telegram.
  4. Показать экономию — бенчмарк + рекомендация КП + публичные цифры.
  5. Не лезть в тендеры — быть «быстрым инженерным карманом» для снабженца.

  Измеримый успех через 12 месяцев:
  • 500+ claimed поставщиков с ≥1 КП
  • 200+ закупок/мес с чертежом
  • median time-to-first-KP < 48 ч
  • 30% repeat rate заказчиков
  • топ-3 в Google по «прямые закупки РТИ» / «закупки нефтесервис чертеж»
