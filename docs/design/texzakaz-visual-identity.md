# ТехЗаказ — визуальная идентичность v3

Статус: **черновик плана, код не писался.** Ждёт «ок» перед Фазой 2.

## Эстетическое направление: «Чертёжный цех»

Из трёх вариантов (чертёжный цех / нефтесервисный пульт / заводская маркировка)
выбран **чертёжный цех**, не смешан с двумя другими.

Почему именно этот, а не два других:
- **Нефтесервисный пульт (SCADA)** отклонён — потребовал бы плотных
  статус-индикаторов и «живых» данных по всему интерфейсу; на практике это
  означало бы переписывать структуру `.proc-row`/KPI-карт, что запрещено
  Hard Rules. Слишком близко к шаблону «enterprise dashboard».
- **Заводская маркировка** отклонена как самостоятельный тезис — её приёмы
  (оранжевый угол, навy/white) уже используются точечно на лендинге и в логотипе,
  но это акциденция бренда, а не цельная история. Если сделать на ней сигнатуру,
  получится просто «оранжевый угол на всём» — декоративно, не информативно.
- **Чертёж** выигрывает, потому что это буквально центральный артефакт продукта:
  заказчик **загружает чертёж**, платформа уже хранит и показывает превью PDF/DWG,
  а верификация компаний уже выражена двумя бейджами
  (`verified_egrul`, `verified_by_platform`) — то есть штамп «ПРОВЕРЕНО» не
  выдуманная метафора, а прямое визуальное прочтение существующей фичи.

## Что уже есть в проекте и используется как материал (не выдумано заново)

- Тёмно-синий `#071B2A` / `#0B2233` уже используется как фон «инженерных» блоков
  на лендинге (`.lp-industrial`, `.lp-scene-hub`, `.lp-faq`).
- Голубой `#0B8FCE` уже встречается как hardcoded-цвет в двух местах
  (`.lp-cta-demo`, `.ai-tz-panel` градиент кнопки) — нигде не объявлен как токен,
  но уже читается как «второй, технический» акцент рядом с оранжевым. В чертеже
  это естественно: **синяя линия чертежа + оранжевая/красная печать штампа**.
  План — формализовать его в токен, а не изобретать новый цвет.
- JetBrains Mono уже используется для номеров заявок, KPI-значений, лейблов
  («ПРЯМЫЕ ЗАКУПКИ · БЕЗ ПОСРЕДНИКОВ») — то есть «техническая аннотация моноширинным
  шрифтом» уже есть как привычка, просто не систематизирована.

## Token plan (Фаза 2 — добавление, не замена)

### Палитра — 6 именованных hex
| Имя | Hex | Роль |
|---|---|---|
| `--tz-stamp-orange` | `#FF6A00` | Primary accent — **без изменений**, основной CTA/бренд |
| `--tz-ink-navy` | `#071B2A` | Тёмные индустриальные блоки, текст высокого контраста |
| `--tz-blueprint-blue` | `#0B8FCE` | Вторичный «чертёжный» акцент: AI/умные подсказки, demo, линии чертежа |
| `--tz-paper` | `#F7FAFC` | Фон «листа» — сетка/чертёж поверх него (уже `--bg-secondary`) |
| `--tz-graphite` | `#475569` | Линии, размерные засечки, подписи (уже `--text-tertiary`) |
| `--tz-verified-green` | `#059669` | Состояние «проверено/принято» (уже `--accent-green`) |

4 из 6 — переиспользование существующих значений под новым семантическим именем
(не новые цвета в палитре, а названные роли поверх неё). Новых по факту два:
`--tz-blueprint-blue` (промотать из хардкода в токен) и формализация `--tz-paper`.

### Типографика
Шрифты **не меняются** (Manrope / JetBrains Mono, self-hosted, хард-требование).
Систематизируем шкалу вместо добавления новой:
- Display (H1 лендинга): Manrope 800, clamp(32px,3.6vw,64px), letter-spacing -1.5px — как сейчас на `#lp-h1`, без изменений.
- Section H2/H3 (кабинет): Manrope 700/800, 16–22px — без изменений.
- Body: Manrope 500/600, 13–16px — без изменений.
- **Annotation/Mono (новая системная роль, не новый шрифт):** JetBrains Mono 600,
  10–11px, letter-spacing 1.5–2px, uppercase — для «размерных» подписей, eyebrow,
  тех. меток. Уже есть как разовый приём — фиксируем как переиспользуемый класс
  `.tz-mono-tag`, чтобы не плодить inline-стили.

### Spacing / radius
`--radius-card` (16px), `--sidebar-width`, `--input-height` и т.д. — **не трогаем**.
Сигнатурный элемент не требует новых радиусов, он работает поверх существующей
сетки как декоративный слой (border/pseudo-element), а не замена форм-фактора.

## Signature element

**Угловые «крепёжные» метки чертежа + штамп «ПРОВЕРЕНО».**

Один визуальный приём в двух применениях:
1. **Corner ticks** — тонкие L-образные засечки (как метки обрезки/регистрации на
   чертеже) в углах одного избранного блока на экран (hero-визуал лендинга,
   sticky detail-панель закупки) — не на каждой карточке. CSS `::before/::after`,
   не требует изменения HTML.
2. **Stamp badge** — `verified_egrul` / `verified_by_platform` бейджи получают вид
   лёгкого «оттиска печати»: двойная окружность/рамка, едва заметный поворот (-2deg),
   moноширинный uppercase текст. Уже существующие DOM-узлы бейджей, меняется только
   CSS.

Используется точечно (hero, верификация, возможно — заголовок sticky-панели
сделки), не на всех карточках — это и есть «одна смелость», вокруг тихо.

## Anti-patterns — сознательно не делаем
- Кремовый фон + serif + терракота; чёрный + кислотный неон — мимо, не наш брифинг.
- Glassmorphism на каждом блоке. **Внимание:** `.lp-faq-panel`/`.lp-faq-card` на
  лендинге уже используют `backdrop-filter: blur` + полупрозрачный белый на navy —
  это ровно тот шаблонный приём, который бриф просит не плодить. В Фазе 3 заменим
  на непрозрачную «бумажную» карточку с лёгкой сеткой вместо блюра (не блюр →
  чертёжная фактура).
- Три одинаковых stat-card в hero (1200+ / 0% / 48ч). **Уже есть на лендинге**
  (`#lp-stats`, landing.html:785-798) — ровно anti-pattern из брифа. План Фазы 3 —
  заменить на единую горизонтальную «штамп-полосу» (см. Hero B ниже).
- Бессмысленные 01/02/03. Существующая последовательность
  Заказчик → ТехЗаказ → КП (`#lp-steps`) — оставляем как есть, это реальный
  процесс с порядком, не декоративная нумерация; не превращаем её в номерные кружки.
- Новые шрифты/Tailwind/UI-библиотеки — не используются (запрещено бриф).

## Hero — 2 варианта (ASCII), выбор

### Hero A — «Рамка чертежа» (раскадровка как сейчас, новая рамка)
```
┌inline crop-marks──────────────────────────────────┐
│ ⌐                                              ⌐  │
│  ПРЯМЫЕ ЗАКУПКИ · БЕЗ ПОСРЕДНИКОВ                  │
│  Прямые закупки                                    │
│  без посредников                                   │
│  [Разместить чертёж] [▷ Демо] [Как это работает]   │
│  ┄┄┄┄┄┄┄┄┄┄┄ (dimension line, "от чертежа до КП")  │
│ ⌐                                              ⌐  │
└─────────────────────────────────────────────────┘
   правая колонка — без изменений (industrial-сцена остаётся)
```
Минимальный риск: меняем только рамку и заменяем `#lp-stats` на размерную линию-
подпись. Хero-сцена справа (`.lp-scene`, `.lp-industrial`) не трогается.

### Hero B — «Штамп-полоса вместо трёх карточек»
```
[Разместить чертёж] [▷ Демо] [Как это работает]

──────────────────────────────────────────────────────
 ТЕХЗАКАЗ-01  ПОСТАВЩИКОВ 1200+  НАЦЕНКА 0%  КП ЗА 48Ч
──────────────────────────────────────────────────────
  (одна моноширинная "штамп"-строка вместо 3 карточек)
```
Заменяет именно anti-pattern (3 одинаковые карточки) на title-block-подобную
строку — как штамп в углу чертежа (название/масштаб/дата одной строкой).

### Выбор: **A + B вместе, но как один диф**
Это не два конкурирующих варианта дизайна, а два независимых исправления одного
hero: A — рамка (контекст всего блока), B — конкретная замена `#lp-stats`.
Беру оба, т.к. B без A теряет смысл (рамка — это то, что делает штамп-полосу
прочитываемой как «чертёж», а не просто как ещё один div). Если нужно сузить
до одного диффа за фазу — первой пойдёт B (прямой анти-паттерн фикс), A — вторым
проходом.
