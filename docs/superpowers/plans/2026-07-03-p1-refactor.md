# P1 Refactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Вынести крупные группы inline-роутов из server.js в routes/*, убрать теневые дубли showToast на фронте. Поведение API не меняется — чистый move-рефакторинг.

**Architecture:** Каждая группа — фабрика `createXRouter(deps)` по образцу существующих routes/*.js; монтирование `app.use('/api/<prefix>', createXRouter(routesDeps))` в зоне строк ~1223-1229 server.js. `routesDeps` расширяется недостающими зависимостями. Требования модулей (exceljs, pdf-билдеры) переезжают в файлы роутеров.

**Tech Stack:** Node/Express 5, без новых зависимостей.

## Global Constraints

- Поведение эндпоинтов НЕ меняется: URL, методы, статусы, тела ответов идентичны.
- Код роутов ПЕРЕНОСИТСЯ (copy-move), не переписывается. Разрешено только: `app.get('/api/x/...'` → `router.get('/...'`, замена замыкающих переменных на поля deps.
- После КАЖДОЙ задачи: `node --check server.js && node --check routes/<new>.js && npm run check && node scripts/pdf-smoke.js`.
- Ветка `feature/p1-routes-refactor`, мерж в main одним куском после всех задач.
- Rate-limiter'ы (`aiLimiter` на строках 257-259, `generalLimiter` 248) остаются в server.js — они app-level и срабатывают до роутеров.
- Роуты health/company-photos/push/telegram/registry-optout (строки 430-1160) НЕ трогать в этом заходе.
