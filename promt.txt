# Задача: визуальная идентичность ТехЗаказ — уникально, не «AI-дизайн», без поломки проекта

## Обязательно
Перед работой активируй skill **frontend-design** и следуй его процессу:
1. Сначала design plan (палитра, типографика, layout, signature element)
2. Self-critique: убери всё, что похоже на типичный AI-шаблон
3. Только после одобрения плана — код

## Проект
**ТехЗаказ (texzakaz.ru)** — B2B-платформа прямых закупок для нефтесервиса России.
Заказчики размещают ТЗ, поставщики подают КП. Аудитория: инженеры, снабженцы, директора заводов — не стартап-тусовка.

**Стек UI (НЕ МЕНЯТЬ):**
- Vanilla HTML + JS (без React/Vue)
- Глобальные стили: `assets/theme-v2.css` (~3200 строк)
- Лендинг: `landing.html` (свои inline-стили + классы `lp-*`)
- Шрифты уже self-hosted: **Manrope** (UI), **JetBrains Mono** (цифры, цены)
- Тёмная/светлая тема через `[data-theme="dark"]` и CSS-переменные
- Иконки: SVG Feather-style, **emoji запрещены**
- Бренд-цвет: оранжевый `#FF6A00`, тёмный navy `#071B2A` на лендинге

## Цель
Сделать визуал **узнаваемым и «индустриальным»** — как инструмент закупщика на заводе, а не как очередной SaaS из 2024. Должно выделяться на фоне конкурентов и шаблонов, но **не ломать** текущую вёрстку, JS, API, модалки, сайдбар, mobile bottom nav.

## Жёсткие ограничения (HARD RULES)

### Нельзя
- Переписывать HTML-структуру страниц кабинета (sidebar, header, `.proc-row`, модалки, таблицы)
- Менять id элементов, onclick, data-атрибуты, inline JS в HTML
- Подключать Google Fonts, Tailwind, Bootstrap, новые npm UI-библиотеки
- Менять логику `assets/app.js`
- Делать full redesign всех 22 HTML за один проход
- Использовать emoji
- Типичный «AI-slop»:
  - кремовый фон #F4F1EA + serif + терракота
  - чисто чёрный фон + кислотно-зелёный акцент
  - бессмысленные 01/02/03 везде
  - purple gradients, glassmorphism на каждом блоке
  - одинаковые rounded cards с одинаковой тенью
  - Inter, Roboto, Space Grotesk «потому что модно»
  - hero с тремя одинаковыми stat-cards (1200+ / 48ч / 0%)

### Можно и нужно
- Расширять `:root` и `[data-theme="dark"]` **новыми CSS-переменными** (tokens)
- Добавлять новые классы в `theme-v2.css` (префикс `tz-` или блоки в конце файла)
- Полировка **landing.html** — отдельный слой `landing.css` или блок в конце landing
- Один **signature element** на весь бренд (например: «чертёжная сетка», штамп ГОСТ, промышленная маркировка, оранжевый угол логотипа)
- Микро-анимации с `prefers-reduced-motion`
- Улучшить: кнопки, карточки, empty states, AI-панель, онбординг, демо-модалку на лендинге

## Эстетическое направление (выбери ОДНО и обоснуй)

Вдохновение из мира заказчика, не из Dribbble:
- **Чертёжный цех** — тонкая сетка, размерные линии, моноширинные метки, штампы «ПРОВЕРЕНО»
- **Нефтесервисный пульт** — плотная информация, чёткая иерархия, статусы как на SCADA (но чисто)
- **Заводская маркировка** — оранжевые углы, промышленные таблички, контраст navy/white

Не смешивай все три. Один визуальный тезис — signature.
