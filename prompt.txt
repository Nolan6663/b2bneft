# Дизайн-ревамп ТехЗаказ — инструкция для Cursor

## Контекст проекта

Это B2B платформа для нефтесервисного рынка (texzakaz.ru). MPA (Multi-Page Application),
без фреймворков. Стек: Node/Express + PostgreSQL, чистый HTML/CSS/JS.

Главный CSS-файл: `assets/theme-v2.css`
Главный JS-файл: `assets/app.js`
Страницы: `index.html`, `producer.html`, `catalog.html`, `partners.html`, `analytics.html`, и др.

---

## КРИТИЧЕСКИ ВАЖНО — ЧЕГО НЕ ТРОГАТЬ

1. **Не менять JS-логику** — только CSS и HTML-атрибуты/классы в разметке
2. **Не переименовывать CSS-классы** — только добавлять новые стили к существующим
3. **Не трогать переменные** — все цвета через CSS-переменные (`:root { ... }`), не хардкодить hex
4. **Не трогать структуру HTML** — не переставлять элементы, не удалять атрибуты
5. **Не трогать `assets/app.js`** — только если нужно добавить CSS-класс в уже существующий элемент
6. **Версию CSS не менять вручную** — `?v=8` в html останется, это кэш-бастинг
7. **Sidebar (`.sidebar`) не переделывать** — только точечные улучшения стилей

## ЧТО НУЖНО СДЕЛАТЬ

### 1. Карточки — глубина и "воздух"

Текущие карточки (`.card`, `.section`, `.kpi-card`, `.req-card`) плоские и однородные.
Нужно добавить им характер через тени и hover-эффекты.

**В `assets/theme-v2.css`:**

```css
/* Ambient shadow для всех карточек */
.card, .section {
  box-shadow: 0 1px 2px rgba(15,23,42,.04), 0 4px 12px rgba(15,23,42,.06);
  transition: box-shadow .2s ease, transform .2s ease;
}
.card:hover, .section:hover {
  box-shadow: 0 2px 4px rgba(15,23,42,.06), 0 8px 24px rgba(15,23,42,.1);
  transform: translateY(-1px);
}

/* KPI карточки — выраженный hover */
.kpi-card {
  transition: box-shadow .2s, transform .18s;
}
.kpi-card:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 28px rgba(15,23,42,.12);
}
```

### 2. Таблицы — промышленный стиль, читаемость данных

Текущие таблицы (`.orders-table`, `.proc-row`) выглядят как дефолтный Bootstrap.
Нужен "dense data" стиль — как в Bloomberg, Linear, Notion.

**Для `.orders-table`:**

```css
.orders-table {
  border-collapse: separate;
  border-spacing: 0;
}
.orders-table thead th {
  background: var(--inner-bg);
  font-size: 10.5px;
  letter-spacing: .08em;
  text-transform: uppercase;
  color: var(--text-muted);
  font-weight: 700;
  padding: 10px 14px;
  border-bottom: 2px solid var(--card-border);
  white-space: nowrap;
  position: sticky;
  top: 0;
  z-index: 1;
}
.orders-table tbody tr {
  border-bottom: 1px solid var(--inner-border);
  transition: background .1s;
}
.orders-table tbody tr:hover td {
  background: color-mix(in srgb, var(--accent-cyan) 4%, var(--card-bg));
}
/* Числа в таблице — моноширинные, правовыравненные */
.orders-table td:last-child,
.orders-table td.col-price,
.orders-table td.col-num {
  font-family: 'JetBrains Mono', monospace;
  font-size: 12.5px;
  text-align: right;
}
/* Полосатые строки — очень слабо */
.orders-table tbody tr:nth-child(even) td {
  background: rgba(0,0,0,.012);
}
```

### 3. Статусы — точки вместо закрашенных таблеток

Закрашенные `.badge` таблетки везде выглядят дёшево. Лучше работает паттерн dot + текст.

**Добавить в CSS:**

```css
/* Dot-badge — статус с точкой */
.status-pill {
  display: inline-flex;
  align-items: center;
  gap: 5px;
  font-size: 12px;
  font-weight: 600;
  color: var(--text-secondary);
}
.status-pill::before {
  content: '';
  width: 6px;
  height: 6px;
  border-radius: 50%;
  flex-shrink: 0;
}
.status-pill.active::before   { background: var(--accent-green); box-shadow: 0 0 0 3px rgba(16,185,129,.15); }
.status-pill.active           { color: var(--accent-green); }
.status-pill.pending::before  { background: #facc15; }
.status-pill.pending          { color: #ca8a04; }
.status-pill.closed::before   { background: var(--text-muted); }
.status-pill.draft::before    { background: var(--accent-blue); }
.status-pill.rejected::before { background: var(--error); }
.status-pill.rejected         { color: var(--error); }
```

Применить классы `status-pill active/pending/closed` вместо `.badge` там, где статус отображается
в списках/таблицах (в `index.html` и `producer.html` в renderRow функциях — добавить только класс,
не трогать JS-логику).

### 5. Кнопки — строгая иерархия

Сейчас `.btn-secondary` выглядит почти так же заметно, как `.btn-primary`.
Нужно усилить контраст: primary — оранжевый и заметный, secondary — почти невидимый.

```css
.btn-secondary {
  background: transparent !important;
  border: 1px solid var(--card-border) !important;
  color: var(--text-secondary) !important;
  box-shadow: none !important;
  font-weight: 500 !important;
}
.btn-secondary:hover {
  border-color: var(--accent-cyan) !important;
  color: var(--text-primary) !important;
  background: var(--inner-bg) !important;
}

/* Ghost кнопки — ещё тише */
.btn-ghost {
  background: transparent;
  border: none;
  color: var(--text-secondary);
  font-size: 13px;
  font-weight: 500;
  cursor: pointer;
  padding: 6px 10px;
  border-radius: 7px;
  transition: background .15s, color .15s;
}
.btn-ghost:hover {
  background: var(--inner-bg);
  color: var(--text-primary);
}
```

### 7. Скелетон-загрузка (shimmer)

Добавить в CSS анимацию для состояния загрузки (применять класс `.skeleton` на элементы
пока данные грузятся):

```css
@keyframes shimmer {
  0%   { background-position: -400px 0; }
  100% { background-position: 400px 0; }

.skeleton {
  border-radius: 6px;
  background: linear-gradient(
    90deg,
    var(--inner-bg) 25%,
    var(--card-border) 50%,
    var(--inner-bg) 75%

  animation: shimmer 1.4s ease infinite;
  color: transparent !important;
  pointer-events: none;
  user-select: none;
}
.skeleton * { visibility: hidden; }
```
